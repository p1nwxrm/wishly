ПОТОМ УДАЛИТЬ!!!!!!!!!!!!!


Папка alembic створюється автоматично при ініціалізації інструменту міграцій. В ній будуть зберігатися файли з історією змін твоєї бази даних. Якщо ти згодом вирішиш додати нове поле до таблиці вішлистів, Alembic згенерує тут спеціальний скрипт, який оновить структуру бази без втрати існуючих даних.

Усі вихідні коди програми лежать у папці app. Її серцем є файл main.py. Це точка входу в застосунок. Тут створюється екземпляр FastAPI, налаштовується CORS (щоб фронтенд міг безперешкодно робити запити до бекенду) та підключаються всі маршрути з інших файлів.

Розділ api відповідає за зв'язок із зовнішнім світом. У підпапці endpoints лежать файли з маршрутами (контролерами). Тут описуються самі шляхи, наприклад @app.get("/gifts"), але без складної логіки бази даних. Файл dependencies.py - це місце для патерну Dependency Injection. Саме тут буде жити функція get_db для видачі сесій бази даних, а також функції для перевірки JWT-токенів поточного користувача.

Папка core містить фундаментальні налаштування. У файлі config.py створюється клас, який зчитує системні змінні оточення (посилання на базу даних, секретні ключі). Файл security.py ізолює в собі логіку безпеки - функції для хешування паролів перед збереженням у базу та генерації токенів авторизації.

Робота з даними чітко розділена на три директорії. Папка db містить database.py, де налаштовується підключення SQLAlchemy до MySQL. Папка models зберігає файли з класами таблиць (User, Gift, Wishlist), які ми розбирали раніше. А папка schemas містить Pydantic-моделі. Це дуже важливий шар валідації: тут описуються формати JSON, які застосунок очікує отримати від мобільного клієнта (наприклад, GiftCreate) або відправляє у відповідь (GiftResponse).

Бізнес-логіка винесена у папку crud (Create, Read, Update, Delete). Саме тут реалізується патерн Data Access Object (DAO). Замість того, щоб писати запити до бази прямо в роутерах, ти створюєш тут файли (наприклад, crud_users.py), які містять функції на кшталт get_user_by_email. Контролер просто викликає цю функцію, передаючи їй дані.

Кореневі файли внизу дерева відповідають за розгортання проєкту. Файл .env зберігає секретні паролі локально і ніколи не додається в систему контролю версій Git. requirements.txt фіксує всі встановлені Python-бібліотеки. А Dockerfile містить інструкції для пакування твого сервера в ізольований контейнер, що дозволить легко розгорнути бекенд на будь-якому віддаленому сервері.
